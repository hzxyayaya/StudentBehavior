# 四类分析任务清单与优先级

Last Updated: 2026-03-25

## 1. 文档目的

本文档用于固化赛题口径下的四类分析任务定义、优先级和与当前 Demo 的关系，避免项目在后续开发和答辩中再次滑回“单任务系统”表达。

## 2. 四类分析任务总览

结合赛题直播答疑内容和当前项目能力，V1 阶段将项目分析任务明确为以下四类：

1. 学业风险动态感知与预警
2. 学业轨迹演化与关键行为分析
3. 学生个体画像与群体分层分析
4. 发展方向与去向关联分析

这四类任务共同构成赛题口径中的“四类模式”，其中第一类是主演示任务，第二类和第三类是强支撑任务，第四类是扩展任务。

## 3. 任务优先级定义

## 3.1 P0：学业风险动态感知与预警

定位：
必须完整实现，是项目答辩和 Demo 的主线。

原因：

- 赛题明确要求必须完成
- 最容易形成“预警 -> 解释 -> 干预”闭环
- 现有数据和现有项目结构都最适合支撑这一任务

V1 完成标准：

- 有明确标签定义
- 有风险预测结果
- 有风险分层展示
- 有解释与干预建议

## 3.2 P1：学业轨迹演化与关键行为分析

定位：
重点支撑任务，需要在 Demo 中有明确展示，但不一定独立做成复杂模块。

原因：

- 赛题强调学生 4 年轨迹演化与关键行为分析
- 能强化“过程干预”而不是静态快照的叙事
- 可以直接增强主任务的说服力

V1 完成标准：

- 至少有学期维度趋势结果
- 能展示关键行为变化与风险变化关系

## 3.3 P1：学生个体画像与群体分层分析

定位：
重点支撑任务，是系统可视化和四象限分层的主要承载体。

原因：

- 赛题明确要求个体画像和群体画像
- 是总览页、群体页和个体页的共同基础
- 四象限分层属于该任务的重要实现方式之一

V1 完成标准：

- 有个体画像
- 有群体画像
- 有四象限分层结果
- 能支持从群体到个体下钻

## 3.4 P2：发展方向与去向关联分析

定位：
扩展任务，用于增强项目完整性和开放性。

原因：

- 赛题直播中明确提到就业去向、学习路径、发展方向都可以成为分析任务
- 可以帮助项目体现“不只做学业预警”
- 但该任务不应阻塞主 Demo 打通

V1 完成标准：

- 至少形成一个轻量分析结果
- 在总览页或任务页中有入口或摘要展示

## 4. 与当前 Demo 的关系

当前 Demo 仍以“学业风险动态感知与预警”为主路径，主流程保持为：

总览 -> 群体 -> 预警 -> 个体

这条主路径不变，但需要在系统叙事上补充说明：

- 总览页展示的是四类分析任务中的综合入口
- 当前重点演示的是任务 1
- 任务 2、任务 3 通过趋势分析、画像和群体分层进行支撑
- 任务 4 作为扩展分析结果进行补充展示

## 5. 对开发范围的影响

为了避免范围失控，V1 开发时建议按以下原则推进：

- 任务 1 做完整闭环
- 任务 2 和任务 3 尽量与任务 1 复用数据和页面
- 任务 4 只做轻量展示，不单独扩展为重模块

这意味着：

- 数据清洗与特征工程优先围绕任务 1 搭建
- 趋势分析和四象限分层作为任务 2、任务 3 的主要实现方式
- 发展方向与去向分析可先做统计关联与可视化，不强求复杂预测模型

## 6. 对答辩叙事的影响

在答辩时，建议不要直接说“我们项目做了一个学业预警系统”，而应说：

本项目围绕四类学生行为分析任务展开，其中以学业风险动态感知与预警作为主演示任务，并通过轨迹分析、画像分层和发展方向关联分析共同构成完整的学生行为分析框架。

这样既能保留主线聚焦，又能对齐赛题直播中的官方口径。

## 7. 当前结论

V1 阶段四类分析任务的优先级排序如下：

- P0：学业风险动态感知与预警
- P1：学业轨迹演化与关键行为分析
- P1：学生个体画像与群体分层分析
- P2：发展方向与去向关联分析

后续所有开发、页面设计和答辩表达，都应围绕这个优先级展开，而不是把四类任务做成平均用力的并列模块。
