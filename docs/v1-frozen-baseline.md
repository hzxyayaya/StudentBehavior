## Project Positioning

项目名称：基于多源数据的大学生行为分析与干预模型设计

V1 不是全量校园治理平台，也不是单一预测脚本或展示型大屏。V1 的目标是交付一个可演示、可解释、可下钻的比赛型分析系统，完成从群体洞察到个体干预的闭环。

## Frozen Goals

V1 必须同时满足以下目标：

- 基于多源校园数据构建学生学期级行为特征
- 围绕 4 类分析任务形成统一展示闭环
- 输出 10 个系统分析结果
- 识别不少于 4 类学生行为模式
- 预测综合测评低等级风险
- 输出不少于 3 个维度的风险归因解释
- 生成面向学校管理场景的干预建议
- 支持管理者先看群体、再下钻个体的使用路径

## Frozen Scope

### Analysis Scope

V1 固定围绕以下 4 类分析任务组织：

1. 学业风险动态感知与预警
2. 学业轨迹演化与关键行为分析
3. 学生个体画像与群体分层分析
4. 发展方向与去向关联分析

V1 固定输出以下 10 个系统分析结果：

1. 学生个体画像
2. 学生群体画像
3. 行为模式识别
4. 学业风险概率分层
5. 风险等级预警
6. 关键影响因子解释
7. 个性化干预建议
8. 学期趋势分析
9. 专业/学院群体对比
10. 模型摘要说明

### User Scope

V1 的核心用户是学校管理者，包括学院管理者、学工管理者、教学质量与学生发展相关管理人员。

V1 的首要叙事视角是群体洞察，不是学生自助端。

### Time Granularity

V1 统一按“学期”作为核心分析粒度。任何数据处理、特征构建、趋势计算、标签对齐和页面展示，默认都按学期组织。

### Integration Key

多源数据整合以“学号”作为统一主键。源字段名可以不同，但必须在清洗阶段归一到学生主键。

## Frozen Business Semantics

### Risk Label

V1 核心监督标签冻结为“综合测评低等级风险”。

- 标签来源：综合测评结果中的等级字段
- 建模形式：优先二分类
- 业务含义：低等级样本视为风险样本，非低等级样本视为非风险样本或较低风险样本

具体等级取值映射规则尚未冻结，但“标签来源于综合测评等级字段”这一决策已冻结。

### Feature Framework

V1 统一围绕以下 8 个特征维度组织特征工程、画像展示、聚类解释和报告生成：

1. 学业基础表现
2. 课堂学习投入
3. 在线学习积极性
4. 图书馆沉浸度
5. 网络作息自律指数
6. 早晚生活作息规律
7. 体质及运动状况
8. 综合荣誉与异动预警

### Behavior Pattern Framework

V1 正式采用“行为模式识别 + 群体分层”的表达框架。

- 业务输出应优先使用可理解的行为模式或群体标签
- 允许使用规则分层、聚类分层或阈值分层
- 聚类编号不是业务输出，业务输出必须是可解释的人群标签和对应说明
- 历史实现中的“四象限”只作为兼容性来源，不再作为冻结业务语义

## Frozen Technical Chain

V1 的技术主链路冻结如下：

1. 多源数据接入与清洗
2. 学号统一与学期级聚合
3. 学生行为特征宽表构建
4. 行为模式识别与群体分层
6. 基于综合测评低等级标签进行风险预测
7. 基于特征贡献生成归因解释
8. 基于规则模板和 LLM 润色生成干预建议
9. 通过管理视角页面完成展示与下钻

V1 优先模型路线冻结为：

- 群体识别：规则分层 / 聚类分层均可
- 风险预测：LightGBM
- 归因解释：SHAP
- 干预生成：规则模板 + LLM 润色

## Frozen Page Path

V1 的最小页面闭环冻结为：

1. 登录页
2. 总览仪表盘
3. 群体分析页
4. 风险预警页
5. 学生个体画像页

增强页：

6. 模型训练与结果管理页

页面主路径冻结为：

总览 -> 群体 -> 预警 -> 个体

模型训练页不进入主演示链路，只作为增强说明页。

## Delivery Rules

后续 agent 协作必须满足以下规则：

- 任何页面、接口、模型和报告设计都必须映射到 4 类分析任务之一，并能落到 10 个系统输出之一
- 不得新增脱离主链路的重型后台能力作为 V1 前置条件
- 不得把聚类编号、裸特征名或技术术语直接作为最终业务表达
- 风险、群体、解释、建议四类结果必须能在同一学生维度上串联
- 如果某项能力无法在 V1 内打通全链路，应降级为明确的 Draft，不得伪装为 Frozen

## Assumptions
