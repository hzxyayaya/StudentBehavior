# 实现优先级路线图

Last Updated: 2026-04-08

## 1. 文档目的

本路线图用于给当前项目的下一阶段开发排优先级，避免在功能、答辩材料和演示稳定性之间失焦。

当前项目已经具备完整 Demo 主链路，但仍存在三类高价值缺口：

1. 真实模型训练与评估链
2. 真实去向分析全链路
3. 真实 LLM 在线报告链路

## 2. 总体优先级

推荐顺序：

1. 真实模型训练链
2. 真实去向分析链
3. 真实 LLM 在线报告链

## 3. 为什么这样排序

### 3.1 第一优先级：真实模型训练链

这是当前项目可信度最强的短板。

目前系统虽然已经有：

- 风险等级
- 风险概率
- 模型摘要
- AUC 展示

但模型层仍带有明显的 `stub` 痕迹，因此若在答辩中被追问：

- AUC 怎么来的
- 模型如何训练
- 标签如何定义
- 训练集和测试集如何划分

当前系统说服力不足。

因此，第一阶段必须先把：

- 风险标签
- 训练脚本
- 评估脚本
- 模型摘要产物

打实。

### 3.2 第二优先级：真实去向分析链

当前系统已经有：

- 发展方向与去向关联分析页面
- 对应 API
- 专业与群体方向分析结构

但系统仍显式保留：

- `去向真值暂未接入`

因此第四类任务当前只是结构存在，未真正闭环。

相比从零开始补大模块，这一项因为已有页面与接口骨架，更适合在真实模型链之后快速补齐。

### 3.3 第三优先级：真实 LLM 在线报告链

当前系统报告层已经能输出：

- `report_text`
- `intervention_advice`
- `intervention_plan`

但核心逻辑仍以规则模板为主。

这一项放最后的原因是：

- 最容易失控
- 最依赖外部服务
- 最容易引入不稳定性
- 最不适合作为第一条真实化改造主线

正确做法应是：

- 模板兜底
- 在线 LLM 增强
- 失败可回退

## 4. 路线图分期

### Phase 1：模型可信

目标：

- 真实风险标签
- 真实训练与评估
- 真实模型摘要

输出：

- 训练脚本
- 评估脚本
- 指标产物
- API 模型摘要真实化

### Phase 2：任务完整

目标：

- 去向真值接入
- 去向类别标准化
- 去向统计与关联分析
- 发展分析页去除占位状态

输出：

- 去向数据 loader
- 去向产物
- 接口真实字段
- 页面真实展示

### Phase 3：报告增强

目标：

- 模板报告 + 在线 LLM 增强
- 保留稳定回退链路

输出：

- LLM 调用模块
- 报告来源标记
- 页面报告来源说明

## 5. 不建议插队做的事项

当前不建议优先做：

- 三角色学生端 / 教师端 / 管理端
- 在线数据录入系统
- 继续深挖纯性能优化
- 大规模视觉重做
- 同时并行推进多个高风险主功能

## 6. 每阶段统一交付顺序

建议每阶段都按同样顺序推进：

1. 数据与标签定义
2. 产物生成
3. API 接入
4. 前端展示
5. 测试与文档更新

## 7. 当前建议

如果下一阶段只启动一条主线，建议直接进入：

- **真实模型训练链**

原因：

- 对项目可信度提升最大
- 对答辩追问防御最直接
- 对后续去向分析与报告增强都有支撑作用
