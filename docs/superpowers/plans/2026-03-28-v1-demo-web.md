# V1 Demo Web Implementation Plan

Status: Active

Last Updated: 2026-03-28

## 1. 计划目标

本计划用于执行 `projects/demo-web` 首版前端 Demo，实现当前 V1 的 4 条主链路页面：

- 总览
- 群体分层
- 风险预警
- 学生个体

前端只消费 `demo-api`，不直接读取 `artifacts/`、数据库或 Excel。

## 2. 实现策略

按以下顺序推进：

1. 新建独立前端子项目与路由骨架
2. 冻结 API client、envelope 解析和基础状态
3. 完成共用布局、主题和导航
4. 先做总览页与群体页
5. 再做预警页与个体页
6. 最后做联调、错误态、演示态打磨

## 5. 当前测试覆盖

当前已补到的测试类型包括：

- API client 与契约测试
- 格式化与错误映射测试
- 预警页查询参数工具测试
- 关键页面联动测试

当前已经覆盖的关键链路：

- 总览页专业高风险摘要跳转到同专业高风险预警页
- 学生页返回预警列表时恢复原有筛选上下文

## 6. 当前开发顺序要求

后续继续迭代时仍必须遵守：

1. 不允许绕过 API client 在页面里裸写请求
2. 不允许页面直接假设旧示例值 `2023-1` 或 `20230001`
3. 不允许把 stub 字段包装成“正式模型结论”
4. 不允许让前端直接读取 `artifacts/`、Excel 或数据库

## 7. 完成定义

当前计划已经基本达到以下完成标准：

- `projects/demo-web` 可独立启动
- 能接当前 `demo-api`
- 4 页主链路可演示
- 有基础测试
- 有基础移动端兼容
- 已进入文档同步、PR 审查和提交准备阶段
