# V1 Demo API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在独立子项目 `projects/demo-api` 中实现一个只读 FastAPI 服务，读取 `artifacts/model_stubs` 结果文件，并按冻结的 API contract 提供总览、群体、预警、个体画像、个体报告和模型摘要接口。

**Architecture:** 保持“接口层”边界清晰。`demo-api` 不读取原始 Excel，不训练模型，不写数据库，只从 `model-stubs` 离线产物加载数据，在 service 层完成最小筛选、分页、聚合和格式转换。错误响应与成功响应统一走 contract envelope，后续如果替换为数据库或正式模型结果，优先替换 loaders/services，不推翻对外接口。

**Tech Stack:** Python 3.12+, `uv`, `fastapi`, `uvicorn`, `pydantic`, `pandas`, `pytest`, FastAPI `TestClient`

## 全局约束

- 全程遵守 TDD：先写失败测试，再写最小实现，再跑通过。
- `demo-api` 只读取结果文件，不直接读取原始 Excel 或 `analytics-db` 内部表。
- `demo-login` 只做固定返回，不引入数据库 session 或 JWT。
- 所有成功/失败响应都必须包进统一 envelope。
- `warnings` 的分页与筛选逻辑必须稳定、可重复。
- `profile` 必须解包 `dimension_scores_json`，不能把 JSON 字符串原样透出。
- `quadrants` 必须由 service 层聚合，不在路由函数里堆业务逻辑。
- 缺失产物文件时必须显式报错，不得伪装成空数据。
