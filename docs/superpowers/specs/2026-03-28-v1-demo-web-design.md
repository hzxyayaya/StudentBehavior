# V1 Demo Web Design

Status: Active

Last Updated: 2026-03-28

## 1. 文档目的

本文档用于冻结 V1 前端 Demo 的项目边界、页面结构、接口消费方式和首版交付范围，避免后续开发再次滑回“先做零散页面，再回头补结构”的模式。

## 2. 项目定位

`demo-web` 是一个独立前端子项目，职责只有一件事：

- 消费 `demo-api` 提供的只读分析接口
- 展示 V1 主链路中的四类分析任务入口与主演示任务
- 支撑答辩、联调和后续替换 stub 逻辑后的前端复用

`demo-web` 不承担以下职责：

- 不直接读取 `artifacts/`
- 不直接读取 Excel
- 不直接连接数据库
- 不在浏览器侧重算模型或聚合逻辑

## 3. 项目落点

目标项目目录为：

- `projects/demo-web`

当前实现仍在独立分支：

- 分支：`codex/v1-demo-web`
- 当前 Codex 工作区路径：`.codex-worktrees/v1-demo-web/projects/demo-web`

技术栈冻结为：

- Vue 3
- TypeScript
- Vite
- Vue Router
- TanStack Query for Vue
- 原生 CSS Variables + 组件级样式

不引入重量 UI 框架作为首版基础，不使用现成后台模板。

## 4. 设计目标

首版前端必须满足以下目标：

1. 能完整承接当前 `demo-api` 的 7 个接口
2. 能展示当前 V1 Demo 的 4 条主链路页面
3. 能在视觉上体现“学生行为分析中台 Demo”，而不是通用后台模板
4. 能兼容当前真实联调值：
   - `term`：`2023-2`、`2024-1`、`2024-2`
   - 默认学期：`2024-2`
   - 示例学生：`pjwrqxbj901`
5. 必须明确区分：
   - 当前主演示任务：学业风险动态感知与预警
   - 支撑任务：轨迹分析、画像分层
   - 扩展任务：发展方向与去向关联分析

## 5. 信息架构

V1 前端只做 4 个主页面：

1. 总览页 `/overview`
2. 群体分层页 `/quadrants`
3. 风险预警页 `/warnings`
4. 学生个体页 `/students/:studentId`

另外保留一个最轻登录页：

5. 登录页 `/login`

默认路由行为：

- `/` 重定向到 `/overview`
- 首次进入先走 `demo-login`
- 登录成功后把 demo token 保存在浏览器侧内存和 `sessionStorage`

## 6. 页面职责

### 6.1 登录页

职责：

- 只做演示登录
- 调用 `POST /api/auth/demo-login`
- 不做真实权限控制

交互要求：

- 页面文案明确标注“演示入口”
- 提供固定默认值，用户可以直接点击进入

### 6.2 总览页

对应接口：

- `GET /api/analytics/overview`
- `GET /api/models/summary`

页面目标：

- 作为四类分析任务的综合入口
- 主展示 P0 任务的群体风险态势
- 同时补充趋势和模型摘要

最小展示模块：

- 当前学期切换器
- 总学生数卡片
- 风险分布卡片组
- 四象限分布卡片组
- 专业高风险摘要表
- 风险趋势图
- 模型摘要卡片

### 6.3 群体分层页

对应接口：

- `GET /api/analytics/quadrants`

页面目标：

- 展示四象限分层
- 作为“群体画像与分层分析”的主要承载页

最小展示模块：

- 四象限宫格
- 每个象限的人数
- 平均风险概率
- 象限主导影响因子

### 6.4 风险预警页

对应接口：

- `GET /api/warnings`

页面目标：

- 展示高风险和中风险学生列表
- 支撑预警筛选和下钻

最小展示模块：

- 学期筛选
- `risk_level` 筛选
- `quadrant_label` 筛选
- `major_name` 搜索或筛选
- 预警表格
- 分页

表格字段：

- `student_id`
- `student_name`
- `major_name`
- `quadrant_label`
- `risk_level`
- `risk_probability`

点击学生行后跳转：

- `/students/:studentId?term=...`

### 6.5 学生个体页

对应接口：

- `GET /api/students/{student_id}/profile`
- `GET /api/students/{student_id}/report`

页面目标：

- 展示学生当前风险画像
- 展示趋势、关键影响因子和干预建议

最小展示模块：

- 学生基础卡片
- 风险等级与风险概率
- 维度评分条形图或雷达替代图
- 学期趋势折线图
- 因子解释列表
- 干预建议列表
- 报告摘要文本

## 7. 视觉方向

首版前端不采用通用紫色后台模板，也不使用扁平白底灰框默认风格。

视觉方向冻结为：

- 主视觉语义：行为脉冲、风险波形、学期轨迹
- 背景：暖灰到深蓝的层次渐变，辅以细网格和柔和径向高光
- 强调色：
  - 风险高：珊瑚红
  - 中风险：琥珀橙
  - 低风险：青绿色
  - 象限分层：海军蓝 / 青绿 / 赭橙 / 暗红的分化体系
- 版式：大标题 + 信息仪表卡 + 留白较大的阅读区
- 动效：页内模块渐进出现、切换学期时卡片平滑过渡

## 8. 前端状态模型

首版前端只保留最小状态：

- 当前 `term`
- 当前 `session_token`
- 预警页筛选条件

其他数据一律通过 TanStack Query for Vue 从 API 获取和缓存。

前端不在本地维护业务真相源，不把接口结果复制成另一套本地领域模型。

## 9. API 对齐要求

前端严格对齐 `demo-api` 的 envelope：

```json
{
  "code": 200,
  "message": "OK",
  "data": {},
  "meta": {
    "request_id": "demo-request",
    "term": "2024-1"
  }
}
```

统一规则：

- `code !== 200` 视为错误态
- `404` 展示“当前学期或学生不存在”
- `500` 展示“当前演示产物缺失或不可用”
- 所有页面都必须有加载态、空态和错误态

## 10. 已知真实口径

前端首版必须按当前真实数据联调，不再沿用旧示例。

可用值：

- `term`：`2023-2`、`2024-1`、`2024-2`
- 示例学生：`pjwrqxbj901`

前端默认学期必须设置为：

- `2024-2`

## 11. Stub 边界

前端必须明确当前以下结果仍然是 stub/规则版：

- `risk_probability`
- `risk_level`
- `quadrant_label`
- `top_factors`
- `intervention_advice`
- `report_text`
- `cluster_method`
- `risk_model`
- `target_label`

这些字段可以展示，但不要在页面文案中暗示“正式建模已完成”。

## 12. 首版非目标

当前不做：

- 多角色权限系统
- 可编辑表单
- 数据回写
- 实时推送
- 真正的模型训练控制台
- 复杂图表配置器
- 移动端独立交互体系

移动端只要求基本可浏览，不要求完整操作效率优化。

## 13. 首版验收标准

V1 前端完成标准：

1. 能在本地独立启动
2. 能成功调用当前 `demo-api`
3. 4 个主页面都可访问
4. 总览、群体、预警、个体 4 条主链路可完整演示
5. 能使用真实 `term` 和真实学生样例联调
6. 页面在桌面端完整可用，移动端基本可浏览
7. 不依赖直接读取原始文件或结果产物

## 14. 当前实现补充说明

截至 2026-03-28，`codex/v1-demo-web` 已经实现并验证的关键行为包括：

- 全局 `DemoStatusBar` 统一提示真实 API、默认学期和 stub 边界
- 总览页提供进入预警列表、群体分层和示例学生页的显式入口
- 总览页“专业高风险摘要”可直接下钻到同专业高风险预警列表
- 群体分层页可携带 `term` 和 `quadrant_label` 下钻到预警页
- 预警页支持 `term`、`risk_level`、`quadrant_label`、`major_name` 筛选
- 预警页筛选条件会写回 URL，并在刷新后恢复
- 学生页支持返回预警列表，并保留原有筛选上下文
- 演示登录已对齐真实 `demo-login` 返回结构，仅依赖 `session_token`
