# V1 Delivery Board

Status: Draft

Last Updated: 2026-03-25

## Purpose

本文档把 V1 冻结基线拆成可执行的交付清单，用于排期、协作和验收。

本文档依赖：

- `docs/v1-frozen-baseline.md`
- `docs/v1-agent-contract.md`

## Milestones

### M1. Freeze Inputs

- [ ] 确认 V1 最小数据源集合
- [ ] 确认综合测评等级原始取值
- [ ] 确认“低等级”标签映射规则
- [ ] 确认学期与评价周期对齐规则
- [ ] 冻结学期级特征宽表 schema

Exit Criteria:

- 数据清洗和标签对齐规则可被复用
- 后续建模不再依赖口头约定

### M2. Freeze Model Outputs

- [ ] 冻结 8 个特征维度的指标构成
- [ ] 产出聚类输入特征清单
- [ ] 冻结四象限映射规则
- [ ] 训练 LightGBM 风险模型
- [ ] 产出风险概率和风险等级映射
- [ ] 产出 SHAP 解释输出格式
- [ ] 产出干预建议规则模板骨架

Exit Criteria:

- 同一学生可同时得到群体标签、风险概率、解释和建议

### M3. Freeze Service Contract

- [ ] 冻结总览接口 contract
- [ ] 冻结群体分析接口 contract
- [ ] 冻结风险预警列表接口 contract
- [ ] 冻结学生画像接口 contract
- [ ] 冻结解释与报告接口 contract
- [ ] 冻结训练与模型摘要接口 contract

Exit Criteria:

- 前端页面不再依赖 mock 结构猜测字段

### M4. Build Demo Path

- [ ] 登录页可进入主应用
- [ ] 总览页展示风险分布、四象限分布、组织对比、趋势摘要
- [ ] 群体页支持查看四象限差异和学生下钻
- [ ] 预警页支持按风险等级、四象限、组织维度筛选
- [ ] 个体页展示画像、风险、解释、建议、趋势
- [ ] 模型训练页作为增强项可选接入

Exit Criteria:

- 可以按“总览 -> 群体 -> 预警 -> 个体”完成完整演示

### M5. Final Review

- [ ] 检查 8 个分析成果是否均有交付载体
- [ ] 检查页面术语和文档术语是否一致
- [ ] 检查四象限命名是否统一
- [ ] 检查风险等级规则是否统一
- [ ] 检查所有待确认项是否仍被标注

Exit Criteria:

- 文档、接口、页面、模型表达一致

## Dependency Order

必须按以下依赖顺序推进：

1. 数据规则
2. 特征宽表
3. 模型输出
4. 接口 contract
5. 页面实现
6. 联调验收

任何试图跳过前置冻结项的实现，都必须标注为临时方案。

## Current Priority

当前优先处理：

1. 冻结综合测评低等级映射规则
2. 冻结学期级特征宽表 schema
3. 冻结最小接口 contract

原因：

- 这三项是模型、后端、前端同时依赖的共享前提
- 如果不先冻结，后续 agent 会在不同假设上并行，导致返工

## Risks

- 当前工作区快照未包含可验证的前后端实现目录，已有实现状态无法直接验收
- 标签映射未定会阻塞训练、评估和页面文案统一
- 最小数据源集合未定会阻塞特征表冻结

## Assumptions

- 现阶段允许先以文档冻结方案，再由实现跟进。
- 若已有历史实现存在，与本文冲突时需要回到基线重新比对。

## Open Questions

- 是否已有可复用的清洗脚本与训练脚本，待确认。
- V1 是否需要显式数据库设计文档，待确认。
- 模型训练页是否进入首版演示，待确认。

## Out of Scope

- 细粒度 Sprint 估时
- 个人分工排班
- 生产运维清单
