# 风险标签定义方案

Last Updated: 2026-04-08

## 1. 文档目的

本方案用于给真实风险模型训练链提供统一、可复现、可答辩的监督标签定义依据。

目标不是一次把所有标签都做复杂，而是先固定一版：

- 业务可解释
- 数据可落地
- 训练可复现
- 答辩可说明

## 2. 标签定义原则

### 2.1 业务优先

标签必须能和学校管理语境对齐，不能只是抽象数学分箱。

### 2.2 学期粒度

统一按“学生-学期”粒度打标签，不混用日级、课程级和毕业级标签。

### 2.3 可追溯

每个标签必须能追溯到真实字段和规则版本。

### 2.4 先简后繁

第一版优先做：

- 二分类风险标签
- 四档风险等级标签

而不是一开始就做复杂多任务联合标签。

## 3. 推荐标签体系

### 3.1 标签一：二分类风险标签

字段名建议：

- `risk_label_binary`

含义：

- `1` = 风险学生
- `0` = 非风险学生

### 3.2 标签二：风险等级标签

字段名建议：

- `risk_label_level`

含义：

- `高风险`
- `较高风险`
- `一般风险`
- `低风险`

## 4. 推荐打标字段

当前最适合作为打标依据的字段包括：

- `term_gpa`
- `failed_course_count`
- `borderline_course_count`
- `failed_course_ratio`
- 综合测评等级或综合测评分位
- 学籍异动类信号

## 5. 推荐第一版规则

### 5.1 二分类标签规则

建议第一版采取保守、容易解释的规则：

- 若挂科门数达到高风险阈值，则 `risk_label_binary = 1`
- 或 GPA 低于风险阈值，则 `risk_label_binary = 1`
- 或边缘课程数达到阈值，则 `risk_label_binary = 1`
- 或综合测评处于明显低等级，则 `risk_label_binary = 1`
- 否则为 `0`

### 5.2 四档标签规则

建议由多个信号加权后映射到四档：

- 高风险
  - 多项核心指标同时异常
- 较高风险
  - 一项强异常 + 若干弱异常
- 一般风险
  - 有风险波动，但未进入强预警
- 低风险
  - 整体稳定

## 6. 版本化要求

标签字段旁必须保留：

- `label_source`
- `label_rule_version`

推荐示例：

- `label_source = academic_rule_v1`
- `label_rule_version = 2026-04-risk-v1`

## 7. 训练与展示的关系

### 7.1 训练优先使用二分类

第一版真实训练建议先用：

- `risk_label_binary`

原因：

- 样本量压力更小
- 指标更稳定
- 更容易先拿到可信 AUC

### 7.2 页面可继续展示四档

即使训练先做二分类，页面仍然可以展示四档风险等级。

做法：

- 二分类模型输出风险概率
- 再根据概率阈值映射到四档风险等级

## 8. 当前建议

真实模型链启动前，必须先把这一份标签规则文档固定，再开始训练。

否则后续会出现：

- 训练标签和页面口径不一致
- AUC 不知道对应什么任务
- 答辩时无法解释标签来源
